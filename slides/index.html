01

Prompt Engineering for GitHub Copilot

Learn how to write prompts that produce consistent, accurate results.

What Is a Prompt?

A prompt is any request you make to GitHub Copilot.

Inline suggestions

Ghost text that appears as you type in the editor.

Copilot Chat

Natural language questions in the Chat panel or inline editor.

Source: Prompt engineering for GitHub Copilot Chat

Six Prompt Engineering Strategies

#StrategyKey idea
1Start general, then get specificState the goal; then add constraints
2Give examplesShow expected inputs and outputs
3Break tasks into stepsCompose features from smaller prompts
4Avoid ambiguityName functions and files explicitly
5Indicate relevant codeOpen only relevant files; use #file
6Experiment and iterateFollow up in the same conversation

Strategy 1 in Practice

Vague 
Write a function to
validate input

Copilot makes assumptions. The result may not match your needs.

Specific 
Write a JavaScript function
that validates an email address.

The function should:
- Accept a single string `email`
- Return true if the format matches
- Return false otherwise
- Use no external libraries
02

Repository-Wide Custom Instructions

Give Copilot persistent, project-specific context for every request.

What Is copilot-instructions.md?

  • A plain Markdown file at .github/copilot-instructions.md
  • Applies to every Copilot Chat request in the repository
  • Provides context you would otherwise repeat in every prompt
  • Supported in VS Code, Visual Studio, JetBrains, and GitHub.com
These instructions do not apply to inline completion suggestions (ghost text).

What to Include

Good to include

  • Language and runtime versions
  • Code style rules (indentation, quotes, semicolons)
  • Framework and library choices
  • Error handling patterns
  • Test runner and conventions
  • File naming conventions

Avoid

  • Instructions that reference specific code paths (use path-specific files for that)
  • Conflicting instructions
  • Very long instructions (they are included in every request)

Verifying Your Instructions

1
Ask Copilot Chat a question that should trigger your instructions.
2
Click Used n references at the top of the response.
3
Confirm that .github/copilot-instructions.md appears in the list.
If the file does not appear, confirm the file path is exactly .github/copilot-instructions.md (case-sensitive).
03

Path-Specific Instructions

Apply targeted rules to files that match a specific path pattern.

How Path-Specific Instructions Work

  • Files live in .github/instructions/
  • File names end with .instructions.md
  • A YAML front matter block specifies the glob pattern
  • When a matching file is open, both instruction sets are active
---
applyTo: "**/*.py"
---

# Python File Instructions

- Follow PEP 8 style conventions.
- Use 4-space indentation.
- Add type annotations to all function arguments.

File Locations

.github/
  copilot-instructions.md          <-- repository-wide
  instructions/
    python.instructions.md         <-- applies to *.py
    tests.instructions.md          <-- applies to *.test.js
    migrations.instructions.md     <-- applies to db/migrations/*

Multiple path-specific files can be active simultaneously if multiple patterns match.

04

Chat Participants, Commands, and Variables

Use special keywords to make your prompts precise and efficient.

Three Types of Keywords

KeywordNamePurpose
@ Chat participant Scopes the prompt to a domain expert
/ Slash command Triggers a specific, common action
# Chat variable Attaches specific context to the prompt

Chat Participants

@workspace

Searches all files in your current VS Code workspace.

@workspace Where is the
user authentication logic?

@github

Accesses GitHub-specific features and web search.

@github What skills are
available?

Type @ in the chat prompt box to see all available participants.

Slash Commands

CommandAction
/explainExplains the selected code in plain language
/fixSuggests a fix for a bug in the selected code
/testsGenerates unit tests for the selected code
/docAdds documentation comments to the selected code
/newScaffolds a new file or project structure

Type / in the chat prompt box to see all available commands.

Chat Variables

VariableAttaches
#fileA specific file from the workspace
#selectionThe currently selected code in the editor
#codebaseSearches across all workspace files
 
#file:starter.py Write unit tests for
the `calculate_average` function.
05

Agent Instruction Files

Onboard AI coding agents so they can work efficiently without extensive exploration.

What Is AGENTS.md?

  • A plain Markdown file that AI coding agents read before starting work
  • Can be placed anywhere in the repository; the nearest file takes precedence
  • Used by the GitHub Copilot coding agent for autonomous tasks
  • Complements copilot-instructions.md (which is for interactive Chat)

AGENTS.md vs copilot-instructions.md

Aspectcopilot-instructions.mdAGENTS.md
Used byCopilot Chat (interactive)AI coding agents (autonomous)
Location.github/ onlyAnywhere in the repo
ScopeRepository-wideNearest file in tree
PurposeGuide chat responsesOnboard agents efficiently

What to Include in AGENTS.md

  • What is this project?
  • How do I build it?
  • How do I run tests?
  • How do I run the linter?
  • What conventions apply?
  • What must I avoid?
  • Are there tricky areas?
  • What is the directory structure?
Target length: approximately 60-150 lines (about 1-2 pages). The agent reads this before any other exploration.
